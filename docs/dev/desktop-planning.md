# Desktop-клиент — планирование

## Windows-клиент

Идея: клиент для Windows по аналогии с Google Disk — иконка в трее + директория в проводнике, работающая через SFTP.

### Как устроены конкуренты

```
Google Drive   → kernel driver (gdfs.sys)       — максимум возможностей
Dropbox        → user-mode + kernel minifilter   — гибрид
OneDrive       → встроен в Windows, CF API       — самый глубокий доступ
Яндекс Диск   → Shell Namespace Extension        — проще всего
sshfs-win      → WinFSP (user-mode FS)           — open source путь
```

**Google Drive** — собственный kernel-mode драйвер (`gdfs.sys`). Файлы существуют в проводнике без скачивания, скачиваются on-demand при открытии. Требует подписи Microsoft, сложнейшая разработка.

**Яндекс Диск** — Shell Namespace Extension (COM-объект, встраивается в проводник как виртуальная папка). Без kernel-драйвера.

### Варианты реализации

| Вариант | Сложность | Суть |
|---|---|---|
| **WinFSP** | средняя | open-source, монтирует как настоящий диск (`Z:\`), user-mode |
| **Dokan** | средняя | аналог WinFSP, чуть старше |
| **WebDAV-прокси** | низкая | приложение поднимает локальный WebDAV поверх SFTP, Windows монтирует нативно |
| **Просто синк** | низкая | синхронизация папки как Dropbox, не виртуальная FS |
| **Kernel driver** | очень высокая | как Google Drive, требует подписи MS |

### Рекомендованный путь

**WinFSP** — потолок разумной сложности для этой задачи. Работает в user-mode, не нужна подпись драйвера, активно поддерживается.

Готовый референс: [sshfs-win](https://github.com/winfsp/sshfs-win) — ровно эта идея, написан на C поверх WinFSP. Можно форкнуть и добавить свой UI.

**WebDAV-прокси** — если нужно быстро: приложение коннектится к SFTP и слушает локальный WebDAV, Windows монтирует `\\localhost\owo` без установки драйверов. Минус — нет офлайн-режима.

### Составные части

- **Трей + UI** — Tauri (предпочтительно, без Electron), C#/WinForms как fallback
- **SFTP** — готовые библиотеки: `SSH.NET` (C#), `libssh2` (C/C++)
- **Виртуальная FS** — сложная часть, см. варианты выше

### Оценка трудозатрат

- С нуля на C# + WinFSP: ~2–4 недели до рабочего прототипа
- Форк sshfs-win + свой трей/UI: несколько дней

## UI Kit для десктопа

Если в будущем потребуется собственный UI kit для десктопных приложений:

- **Tauri** — WebView-based, фронтенд на Svelte/React, CSS custom properties работают без изменений
- Тот же `@owo-ui/css` пакет применим напрямую — никаких адаптаций не нужно
- Tauri поддерживает Windows, macOS, Linux из одного кодобазы
